Run the `job-readiness` skill. For the application pack I name (a slug or a
directory), show me its readiness score against my master CV: the overall 0–100
number, the pass/warn/fail checklist for each factor (fabrication gate first, then
ATS, fit, completeness, red flags), and honest next steps. If a keyword is one I
already have but it isn't in the tailored CV, tell me to re-tailor to surface it;
if it's a genuine gap, point me to /job-upskill — never suggest inventing it. If
the fabrication check failed, tell me plainly it is not ready and why. This runs
before /job-apply so I can strengthen the application honestly first.
